Data Preperation

SkillsUSA Presentation Project


The spread sheet used as the starting data source for the presentation is divided into two sections:

       Sheet1 - maintains the main Category listing and the image file name that is associated with that Category.

       Sheet 2 - the contestant listing and their related school information. The additional AWARDS and PRIZE fields in Sheet2 are used as reference positions and should be remain blank within the spreadsheet.


Team Events

Sheet2 of the spreadsheet contains a TEAM column where team numbers can be entered for any team events. The team number is displayed in the presentation with the contestants for that team. A team is displayed as a team only if each member of that team have the same corresponding team number. Team contestants without a team number will be displayed in the presentation individually.



When the application first loads the data from the spreadsheet, the category used with the contestant is compared against the categories in the main category list so that the proper image can be associated to that contestant during the presentation. If the category is not found, an error message is generated indicating that not all the values could be loaded.



The image field for the contestants that have a 'missing' category will be displayed in red. In these situations, the category used with the contestant should be checked against the categories in the main category listing.



Possible problems include:

Variations in wording such as Quiz Bowl and Quiz Bowl Alternate

Variations in lowercase and uppercase such as Masonry HELPER and Masonry helper

Additional white space in acronyms such as HVAC and H V A C


These items are easily corrected by adding the variation of the category to the main category listing or adjusting the spelling/case style used in either the category listing or contestant listing. After making any data corrections, just reload the spreadsheet into the application.


In the event that a new category is being used, simply add that category to the category listing and specify the image that should be used for the category.

If using a new image, make sure you also place a copy of the new image in the _images directory of the application. Recommended dimensions for any new images are 117 pixels wide by 115 pixels high.
